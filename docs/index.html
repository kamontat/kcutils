Basic concept

Mono-repository managed by lerna. Typescript is primary language but some packages still using Javascript. We use tsc as primary compiler with optional parcel or rollup can be use on some project as well, jest as primary testing tools with stryker mutator supported, typedoc as documentation website generator, and eslint as primary linter.

User (old)

This is a utilties project for another typescript repositories. This include compiler for typescript, testing framework like jest, frontend like react and code style / linter as eslint. I separate the modules into 2 namespace @kcinternal and @kcutils (+ private).

@kcinternal (old)

This namespace is for project which use internally, all of them should install as devDependencies in package.json.

  • @kcinternal/commandline - commandline tools like compiler, linter, test, and more
  • @kcinternal/config - contains useful configuration for commandline and more
  • @kcinternal/graph - generates dependencies graph for kcmono project

@kcutils (old)

This is a utilities projects for generic usecase

  • @kcutils/error - Error management and custom error object
  • @kcutils/logger - Another logger services
  • @kcutils/helper - Functional utilities like lodash/ramda
  • @kcutils/models - Custom class models for multiple usecase like Optional
  • @kcutils/color - Color utilities and convertion
  • @kcutils/react-testkit - Enzyme support and React utilities function for react
  • @kcutils/testkit - Stryker support with default configuration (Deprecated)
  • @kcutils/gatsby-config - Gatsby Configuration Builder for Typescript project
  • @kcutils/random - Pseudorandom number generators

Private (old)

This is a private project which never publish to npm

  • @kcutils/example - Example project for give idea how to create new module in this repository
  • @kcutils/graph - Application project for generate dependencies for this repository
